### 실행하기 전 설정

1. docker 먼저 다운
2. 관리자 전용 터미널에서 dvwa 등록
    docker run -d --name dvwa -p 8080:80 vulnerables/web-dvwa
3. docker start dvwa



### config 설정

config 파일은 모두 config/ 폴더 아래에 있음.
어떻게 작성하는지는 examples/ 폴더 참고.

config/proxy_config.json — 프록시 설정
config/target_config.json — 진단 대상 설정

프록시를 아예 끄려면 "enabled": false로 설정.



### 실행 순서

1. mitmproxy 실행
    mitmdump --listen-host 0.0.0.0 -p 8081 -s proxy/capture.py

    mitmproxy가 켜져 있는 동안 results/captures/capture_xxx/proxy_history.jsonl에 트래픽이 누적됨.
    mitmproxy를 재시작하면 새로운 capture 폴더가 생성됨.

2. 테스트 진행 (크롤 + merge)
    python test_all.py



### 실행 결과

크롤 실행 후 results/sessions/session_xxx/ 아래에 생성:

    session_meta.json             세션 정보 (target_url, proxy, 시작/종료 시각)
    crawl_result.json             크롤 결과 (guest/member 통합, role 필드로 구분)
    proxy_history_snapshot.jsonl  해당 세션 시간 범위의 프록시 캡처 (target_url 요청만)
    scan_targets.json             크롤 결과 + 프록시 결과를 합쳐 스캔 가능한 부분만 모아놓음
