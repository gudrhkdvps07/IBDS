## 사전 설치
- Docker Desktop 설치 
- cmd에서 docker desktop start로 실행 가능

---

## 1. ZAP 설정 (프록시 / 수집)

### 최초 1회만 실행
```powershell
.\zap_setup.ps1
```
자동으로 다음을 처리함:
- ZAP Docker 이미지 pull
- ZAP 컨테이너 시작 (포트 8081, `0.0.0.0` 바인딩)
- ZAP 초기화 완료 대기
- `zap_root_ca.cer` 인증서를 프로젝트 루트에 저장
- `config/zap_config.json` 자동 생성 (API Key, 포트 포함)

> API Key 및 포트를 변경하려면 `zap_setup.ps1` 상단의 `$API_KEY`, `$ZAP_PORT` 수정 후 재실행

### 이후 실행 시
```powershell
docker start zap
```

### ZAP 종료
```powershell
docker stop zap
```

## 2. Firefox 설정 (최초 1회)

### IP 확인
- cmd 혹은 powershell에서 ipconfig
- 유선랜: 이더넷 어댑터 이더넷에서 ipv4주소 확인
- 무선랜: 무선 LAN 어댑터 Wi-Fi에서 ipv4주소 확인


### 프록시 설정
`about:preferences#general` → 네트워크 설정 → 수동 프록시 설정
- HTTP 프록시: IP: 아까 확인한 본인 IP / 포트: `8081`
- HTTPS도 동일하게 적용 체크

### 인증서 설치
- 프로젝트 루트에 있는 `zap_root_ca.cer` 선택
- "Trust this CA to identify websites" 체크 → OK

## 3. DVWA 설정 (테스트 대상 서버)

관리자 터미널에서 실행:
```powershell
docker run -d --name dvwa -p 8080:80 vulnerables/web-dvwa
```

이미 등록된 경우:
```powershell
docker start dvwa
```

브라우저에서 `http://본인IP:8080` 접속 → Setup 페이지에서 DB 초기화 → admin/password 로그인

> Firefox 프록시를 통해 ZAP에 트래픽을 잡으려면 `localhost` 대신 실제 IP로 접속해야 함 (Firefox는 localhost/127.0.0.1 프록시 우회)

---


## 4. 수집 실행

Firefox로 `http://본인IP:8080` 탐색 (로그인 포함, 여러 페이지 이동)

```powershell
python main_collect.py
```

결과 확인:
- `results/collection/raw_zap_messages_sample.json` — ZAP raw 메시지 샘플
- `results/collection/collected_targets.json` — 변환된 RequestTarget 목록

## 설정 파일

| 파일 | 설명 |
|---|---|
| `config/zap_config.json` | ZAP 연결 정보 (host, port, api_key) — `zap_setup.ps1` 실행 시 자동 생성 |
| `config/target_config.json` | 스캔 대상 URL 및 인증 쿠키 |
