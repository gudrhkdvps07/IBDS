### 실행하기 전 설정

1. docker 먼저 다운
2. 관리자 전용 터미널에서 dvwa 등록
    docker run -d --name dvwa -p 8080:80 vulnerables/web-dvwa
3. docker start dvwa



### config 설정

config 파일은 모두 config/ 폴더 아래에 있음.
어떻게 작성하는지는 examples/ 폴더 참고.

config/proxy_config.json — 프록시 설정
config/target_config.json — 진단 대상 설정

프록시를 아예 끄려면 "enabled": false로 설정.



### 실행 순서

1. mitmproxy 실행
    mitmdump --listen-host 0.0.0.0 -p 8081 -s proxy/capture.py

    mitmproxy가 켜져 있는 동안 results/captures/capture_xxx/proxy_history.jsonl에 트래픽이 누적됨.
    mitmproxy를 재시작하면 새로운 capture 폴더가 생성됨.

2. 테스트 진행 (크롤 + merge)
    python test_all.py

3. 요청기 실행 (tester)
    tester/sample_test_cases.json에 PHPSESSID 등 실제 값을 채운 뒤:
    python tester/main.py

    결과는 results/sessions/session_xxx/test_results.jsonl에 저장됨. 실행할 때마다 덮어씀.
    크롤(2번)을 먼저 실행하지 않으면 세션 폴더가 없어서 에러남.



### 실행 결과

크롤 실행 후 results/sessions/session_xxx/ 아래에 생성:

    session_meta.json             세션 정보 (target_url, proxy, 시작/종료 시각)
    crawl_result.json             크롤 결과 (guest/member 통합, role 필드로 구분)
    proxy_history_snapshot.jsonl  해당 세션 시간 범위의 프록시 캡처 (target_url 요청만)
    scan_targets.json             크롤 결과 + 프록시 결과를 합쳐 스캔 가능한 부분만 모아놓음



### 요청기(tester) 실행

요청기는 test_cases.json을 읽어 HTTP 요청을 보내고 결과를 JSONL로 저장한다.
mutator가 없는 현재는 sample_test_cases.json을 직접 작성해서 사용한다.

#### 사전 준비

1. httpx 설치
    pip install httpx

2. DVWA 로그인 후 세션 쿠키 확인
    브라우저에서 http://localhost:8080/login.php 로그인
    개발자도구 > Application > Cookies 에서 PHPSESSID 값 복사

3. tester/sample_test_cases.json 에서 아래 값을 실제 값으로 교체
    "PHPSESSID": "<복사한 세션값>"
    "User-Agent": "<실제 브라우저 UA 문자열>"
    security 쿠키 값은 DVWA Security 설정과 일치해야 함 (low / medium / high)

#### 실행

    python tester/main_tester.py

결과 파일은 results/sessions/session_xxx/test_results.jsonl에 저장됨. 실행할 때마다 덮어씀.
나중에 변형기가 생기면 tester/main_tester.py 상단의 _CASES_PATH만 교체하면 됨.

#### 실행 결과 확인

콘솔에 케이스별 상태 출력:

    [200]   proxy-manual-001: GET http://localhost:8080/vulnerabilities/sqli/
    [200]   proxy-manual-002: GET http://localhost:8080/vulnerabilities/sqli/
    Done — 2 ok / 0 error / 0 skipped

engine이 http가 아니거나 지원하지 않는 body_type이면 [SKIP]으로 표시되고 JSONL에도 skipped 레코드로 저장됨.

#### skipped / error 레코드 필터링 (참고)

    # skipped만 보기
    python -c "import sys,json; [print(l) for l in open(sys.argv[1]) if json.loads(l).get('skipped')]" result/session/test_results.jsonl

    # error만 보기
    python -c "import sys,json; [print(l) for l in open(sys.argv[1]) if json.loads(l).get('error')]" result/session/test_results.jsonl
